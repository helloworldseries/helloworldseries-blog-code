How to ignore new files or existing files in git

Posted by Arpit Jain on August 09, 2016

From time to time, there are files you don't want Git to check in to GitHub. There are a few ways to tell Git which files to ignore.

Ignore files - globally (.gitignore)

As a developer, I don't want git to contain generated files like *.class, *.a or editor specfic configuration files etc.

All such files (to be ignored), can be added in .gitignore file so that those files will be ignored while tracking.

You would need to commit .gitignore file, so that this ignorance would be benefited to all developers, whoever checks out this repo

  1. Open git bash terminal.

  2. Navigate to C:\MyGITRepo (This directory was created in How to init git repo)).

    cd C:\MyGITRepo

  3. Create an empty HelloWorld.java. 

    touch HelloWorld.java

  4. Create an empty HelloWorld.class (Dummy class, not generated by compiling java file). 

    touch HelloWorld.class

  5. Create new directory structure com/helloworldseries 

    $mkdir -p com/helloworldseries

Ignore files - locally (local to your project only)

  1. Open git bash terminal.

  2. Navigate to C:\MyGITRepo (This directory was created in How to init git repo)).

    cd C:\MyGITRepo

  3. Create a blank file with .gitignore

    touch .gitignore

  5. Before we move on to next tutorial, let's summarize all the commands used in this tutorial. 

    cd /C/MyGITRepo
