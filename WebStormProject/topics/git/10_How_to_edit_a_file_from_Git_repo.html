Edit a file of git repository

Posted by Arpit Jain on August 09, 2016

Multiple ways to edit a file of git repository

Git allows three ways to edit a file using -

  1. Command line (bash terminal or any other client)

  2. git gui (or any other gui client) - will cover in some other tutotial

  3. Directly from Github repository - will cover in some other tutotial.

Let's use bash terminal to edit a file.

  1. Open bash terminal.

  2. Navigate to C:\MyGITRepo\.(This directory was created in How to init git repo).

    cd C:\MyGITRepo

  3. Use your favorite editor to edit a file i.e. helloworldseries.txt and save into C:\MyGITRepo\.

  4. Switch to bash terminal and execute command to commit this file.

    git commit -m "commit message"

  5. You will see some message like "Changes not staged for commit".

  6. Now commit this file using git -am commit.

    git commit -am helloworld.txt "commit message"

  7. Command to push the file(s) - 

    git push <remote-name> <branch-name>

    Replace<remote-name> with origin and <branch-name> with master.

    We are using master branch for all operations. In real life, you would probably use some other branch name and not master.

    So the final command would be..

    git push -u origin master

    -u will prompt for your github userid and password. After successful authentication, all files in committed state in local repo will be pushed to remote tepo.

  8. Let's verify our changes in Our Github repository.

  9. If you have reached to this step, it means that you have successfully edited your first file in github repository.

    It's not a time to relax. Let's move on to next tutorial and see how to delete a file from github repo.

  10. Before we move on to next tutorial, let's summarize all the commands used in this tutorial. 

    cd C:\MyGITRepo
git commit helloworld.txt -m "commit message"
git commit helloworld.txt -am "commit message"
git push -u origin master
