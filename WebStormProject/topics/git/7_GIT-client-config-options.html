How to Customize GIT configuration

Posted by Arpit Jain on August 09, 2016

Why you may want to customize GIT configuration.

  1. Username/email (preferred name like "John Doe" and "John.Doe@example.com") for GIT commits/push/pull etc. instead of logged in user ID.
  2. Editor to create and edit your commit and tag messages core.editor .
  3. Template (subject line, what is the change) for all of your commit and tag messages commit.template .
  4. Pagination option core.pager for Git output such as log and diff .
  5. Ignore certain files core.excludesfile for all repositories that you work with.
  6. Autocorrect help.autocorrect help
  7. Enable/Disable colors in Git terminal color.ui
  8. External Merge and Diff Tools
  9. Formatting and Whitespace core.autocrlf

How to find current git client configurations?

Git reads values from three different configuration files.

  1. system

    Applied to all the users of this machine.

    Configuration file - C:\Program Files (x86)\Git\etc\gitconfig . 

    2. git config --system
  2. global

    Applied to single user.

    Configuration file (~/.gitconfig) . This file can be used to overwrite configuration value set by system configuration file.

    3. git config --list --global
  3. local

    Applied to a single repository

    Configuration file (.git/config) . This file can be used to overwrite configuration values set by global and system configuration files.

    4. git config --list --local

Order of precedence - GIT config

  1. system options are loaded first.
  2. global options are loaded after system options . i.e if provided, it can override system options.
  3. local options are loaded at the end. i.e if provided, it can override system and global options.

Use your favorite username and email.

  • --system

    • git config --system user.name "John Doe"
git config --system user.email "John.Doe@example.com"

  • --global

    • git config --global user.name "John Doe"
git config --global user.email "John.Doe@example.com"

  • --local

    • git config --local user.name "John Doe"
git config --local user.email "John.Doe@example.com"

Above approach can be followed to set all other configuration preferences like core.editor, commit.template, core.pager, core.excludesfile, help.autocorrect, color.ui etc.
